Tutorial: Getting Started with an OAuth2 App
This guide will walk you through the steps of using the API with OAuth2 authentication.

Overview

With this integration, you will be able to access your users' Coinbase accounts via the OAuth 2.0 protocol, without needing their API Key or password. When authenticating a user with OAuth2, you can specify the permissions required, which will limit that token's account access.

OAuth2 enables several use cases:

  • The ability to initiate seamless bitcoin payments from customers, after initial authorization.
  • The ability to access customer's account data, after initial authorization. For example, access to account data could enable you to provide an analytics app for bitcoin spending.
  • The ability to take advantage of Coinbase's full API on behalf of a user. For example, OAuth2 can give a marketplace the ability to manage seller's accounts.

Tutorial: Getting Started With OAuth2

Here's an overview of what you'll complete in this tutorial:

  1. Register an application
  2. Implement the OAuth flow
    • Redirect users from your site to authorization page
    • Obtain access code
    • Initiate POST request to get access token
    • Use access token to make authenticated requests
  3. Handle refresh tokens

Step 1. Register a new OAuth application

Primero necesitarás registrar una solicitud.

You'll need to give your application the following:

  • name: this will be shown to users
  • redirect uri: where the user will be taken to, after authorizing your application.

Podrás editar estos detalles entrando en tu list of applications en cualquier momento.

Once you've set these you'll receive the following:

  • Client ID: a 64-character alphanumeric string specific to your account.
  • Client Secret another 64-character alphanumeric string specific to your account.

Observa: si estás desarrollando una aplicación instalada (como por ejemplo una aplicación de móvil o escritorio), ve a la sección de autenticación para establecer tu URL de redireccionamiento.


Step 2. Implement the OAuth flow

Three-legged Oauth2 requires the user to log into their account and then authorize your app on a browser page.

The example below demonstrates how to complete the functionality from scratch. You can alternatively use a number of OAuth2 libraries, many of which support refresh tokens so you won't have to worry about tham.

There are four key steps to implementing the OAuth functionality. Let's get started!

a. Redirect user from your site to authorization page

The following paths are used for requesting tokens: 

GET     https://www.coinbase.com/oauth/authorize    display authorization form
POST    https://www.coinbase.com/oauth/token        endpoint for requesting tokens

As a first step, redirect your user to the following page, replacing the fields for YOUR_CLIENT_ID and YOUR_REDIRECT_URI with your credentials: 

https://www.coinbase.com/oauth/authorize?response_type=code&client_id=YOUR_CLIENT_ID&redirect_uri=YOUR_REDIRECT_URI

Expected results:

  • The user will first see a login page for Coinbase (if they are not already logged in)
  • Once logged in, the user will see an authorization page for your app

User signs into Coinbase
Authorization screen

b. Obtain access code

If the user authorizes your app, he or she will be redirected to YOUR_CALLBACK_URI?code=CODE

Expected results:

  • A code parameter from YOUR_CALLBACK_URI?code=CODE for your next request

    • c. Initiate POST request to get access token

    Now you've got the code, you'll need to swap this for an access_token.

    Initiate a POST request to get the access token as below. You will need to replace the placeholders with your own credentials for:

    • code
    • YOUR_CALLBACK_URI
    • YOUR_CLIENT_ID
    • YOUR_CLIENT_SECRET 
    POST https://www.coinbase.com/oauth/token?grant_type=authorization_code&code=CODE&redirect_uri=YOUR_CALLBACK_URI&client_id=YOUR_CLIENT_ID&client_secret=YOUR_CLIENT_SECRET

    The expected result is a response containing the access token: 

    {
    "access_token":"...",
    "token_type":"bearer",
    "expires_in":7200,
    "refresh_token":"...",
    "scope":"all"
}

    Note that you will have received a refresh_token as well, which will be useful later on.

    d. Use the access token to initiate an authenticated request

    For example, you can initiate a GET request for the balance of the account: 

    GET https://api.coinbase.com/v1/account/balance?access_token=YOUR_ACCESS_TOKEN

    The expected result is a response containing the balance: 

    {
  "amount": "36.62800000",
  "currency": "BTC"
}

    Step 3. Handle refresh tokens

    Coinbase uses a feature of OAuth2 called refresh tokens. This process complicates the integration, but provides an additional layer of security since a compromised access token is automatically revoked after two hours.

    Here's how the access_token and refresh_token work:

    • The access token expires after two hours, after which you'll need to use a refresh token to obtain a new access token and refresh token
    • The refresh_token never expires but can only be used once.
    • Anytime you get an access token you'll also be given a new refresh_token.
    • If the access token is expired, the results of your requests will be blank - the headers will return with a 401 error.

    To get a new access token, you'll need to do a POST request as before, but this time (1) include your refresh_token and (2) change the grant_type to refresh_token. Also, note that the code parameter will not be needed in this request. 

    POST https://www.coinbase.com/oauth/token?grant_type=refresh_token&redirect_uri=YOUR_REDIRECT_URI&client_id=YOUR_CLIENT_ID&client_secret=YOUR_CLIENT_SECRET&refresh_token=REFRESH_TOKEN

    The expected result is a response containing the access token, as before: 

    {
    "access_token":"...",
    "token_type":"bearer",
    "expires_in":7200,
    "refresh_token":"...",
    "scope":"all"
}

    Congratulations! At this point, you'll have your OAuth2 authentication up-and-running!

    Now it's time to build out the rest of your application. You can access our referencia API completa to explore the different API calls you can make. You may also want to read about parámetros adicionales para OAuth2.
