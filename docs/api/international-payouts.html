Tutorial: Getting Started with International Payouts
This guide will explain how to do international payouts with bitcoin.

“I have a business with international vendors and contractors that are based all over the world. I'd like to pay them in bitcoin, as I think it would be cheaper and faster. Where do I start?"

Overview

It's easy to send bitcoin to people all over the world with Coinbase. You can use bitcoin to pay contractors or vendors, or just to send funds to friends and family.

Bitcoin arrives almost instantly and without hefty foreign exchange fees. Once your bitcoin arrives, the recipient can either keep the funds as bitcoin or, depending on the location, can use a local exchange to sell the bitcoin into their local currency.

To get started, you'll need a Coinbase account. If you don't have one, you can see more details on how to sign up below:

  • Inicia sesión con una cuenta de Coinbase

    You can sign up for a Coinbase account here: https://www.coinbase.com/signup.

    In order to sell the bitcoin that you receive into USD, you'll need to reach what we call "Level 1" verification, by doing the following:

    • Verifying your email address
    • Verifying your phone number
    • Adding a bank account
    • Add a credit card (optional)

    You can view your current verification status and buy limits at https://www.coinbase.com/verifications.


    Sign up for an account
    Reach Level 1 Verification
    Add a credit card for instant-buy (optional)

    The last step, adding a credit card, is optional. The advantage of adding a credit card is that it enables "instant-buy", allowing you to buy up to $100 of bitcoin right away.

Making International Payouts

In this guide, we'll show you two different options for doing international payouts:

  • Using our site (which doesn't require any technical work)
  • Using our API, which enables you to automate the payouts (requires programming)

We'll first go through how to make payouts using our site. We'll then present the more automated route in the API section.

Option 1: Using the Coinbase site to make payouts

1a. Buy bitcoin

The first step is to buy the bitcoin so that you can send it to your payees.

Go to the buy page at https://www.coinbase.com/buys to purchase the bitcoin.

Note: it will take up to 4 business days for bitcoin to arrive in your account, unless you are using our instant-buy feature.


Buy bitcoin

1b. Send bitcoin

Once your bitcoin has arrived, you can send it to the email address or to the bitcoin address of your recipient.

Select the "Send Money" button in the top right-hand-side of your home screen, to bring up a window where you can fill in the amount and the recipient details.


Select "Send Money"
Fill out details of send.

If your recipient does not have a Coinbase account, sending bitcoin to their email address will create a Coinbase wallet associated with their email. The recipient can then log in and claim the bitcoin.

Once you click send, your bitcoin payments will be sent, anywhere in the world, and will arrive almost instantly!

Congratulations, you've made your first payouts in bitcoin!

Option 2: Using our API to automate payouts

You can use use our API to streamline your payouts. This can be useful if you have a large number of payouts, and are looking for an automated solution.

These API requests will require authentication, either using the API Key + Secret method or with OAuth2. With this use case, the API Key + Secret is the simplest solution, which you can read about here:

  • primeros pasos con la autentificación de Clave API + Secreto.

    Step 1. Create an API Key + Secret

    If you haven’t already done so, you will need to sign up for a Coinbase account and to create a Key + Secret pair in your API settings.

    Step 2. Access the API using your Key + Secret

    In this step you will use your Key + Secret pair to access our API. We offer several pre-completed code examples below, where you can simply paste your account’s credentials into the fields available for API_KEY and API_SECRET.

    • Ejemplo de Python 
      import os
import hashlib
import hmac
import urllib2
import time
import json

#This code sample demonstrates a GET call to the Coinbase API
#Before implementation, set environmental variables with the names API_KEY and API_SECRET.

def make_request(url, body=None):
  opener = urllib2.build_opener()
  nonce = int(time.time() * 1e6)
  message = str(nonce) + url + ('' if body is None else body)
  signature = hmac.new(os.environ['API_SECRET'], message, hashlib.sha256).hexdigest()


  headers = {'ACCESS_KEY' : os.environ['API_KEY'],
             'ACCESS_SIGNATURE': signature,
             'ACCESS_NONCE': nonce,
             'Accept': 'application/json'}

  # If we are passing data, a POST request is made. Note that content_type is specified as json.
  if body:
    headers.update({'Content-Type': 'application/json'})
    req = urllib2.Request(url, data=body, headers=headers)

  # If body is nil, a GET request is made.
  else:
    req = urllib2.Request(url, headers=headers)

  try:
    return opener.open(req)
  except urllib2.HTTPError as e:
    print e
    return e

# Example of a GET request, where body is nil.
print make_request('https://api.coinbase.com/v1/account/balance')

    • Ejemplo de PHP

      This is a "plug and play" PHP script for making GET and POST requests to the API. It does so through a single function called coinbaseRequest() which takes three parameters:

      • $directory: the directory to which you're making your request. That is, whatever comes after "https://api.coinbase.com/v1/" in your request URL.
      • $getOrPost: "get" if it's a GET request; "post" if it's a POST request.
      • $parameters: Your API request's GET or POST parameters, formatted as JSON. If you're making a request with no parameters, its value should be "" or '{}'. 

      <?php

function coinbaseRequest($directory,$getOrPost,$parameters){

$ch = curl_init();
//Your API Key and Secret go here.
$apiKey = "API_KEY";
$apiSecret = "API_SECRET";
$nonce = sprintf('%0.0f',round(microtime(true) * 1000000));
$url = "https://api.coinbase.com/v1/" . $directory;
$parameters = @http_build_query(json_decode($parameters), true);

if($getOrPost == "post"){
  curl_setopt_array($ch, array(CURLOPT_POSTFIELDS => $parameters,CURLOPT_POST => true));
}else if($parameters != ""){
  $url .= "?" . $parameters;
  $parameters = "";
}

curl_setopt_array($ch, array(
    CURLOPT_URL => $url,
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => array(
    "ACCESS_KEY: " . $apiKey,
    "ACCESS_NONCE: " . $nonce,
    "ACCESS_SIGNATURE: " . hash_hmac("sha256", $nonce . $url . $parameters, $apiSecret)
    )));

$results = curl_exec($ch);
curl_close($ch);
return $results;
}

?>

    Once you've updated the code with your API_KEY and API_SECRET, you're ready to submit the request! Go ahead and run the script.

2b. Send Bitcoin

You can send bitcoin with the send_money API call, documented aquí en la referencia de API. 

POST https://api.coinbase.com/v1/transactions/send_money

A required parameter is transaction[to], where you can specify an email or a bitcoin address. You'll also need to specify an amount, either by passing an amount parameter, which is interpreted as a bitcoin amount, or by passing an amount_stringand amount_currency_iso parameter such as USD, which will then send an equivalent amount of bitcoin at current exchange rates.

The send_money API call also has an instant_buy parameter, which can be used to purchase the necessary funds first, then send them. This will only work if Instant Buy is enabled on your account and the amount being purchased is less than or equal to the remainder of your daily limit. If you are using this parameter, you could complete both the buy and the send in this one api call.

If the instant buy is successful, the response will come back with an additional transfer field representing the purchase.
