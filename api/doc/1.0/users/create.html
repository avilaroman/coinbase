POST /api/v1/users
Create or signup a new user.

Unauthenticated resource that creates a user with an email and password.

The receive address for the user is returned as well if you’d like to send a first payment to them. To generate additional receive addresses you will need to be authenticated as this user.

Note that if you would like to create a user and send them a payment (using their email address), you can do this in one step by calling the send_money api call. This will create the user and send them a payment, asking them to sign in and claim the amount (they will choose a password on this first step). If they don’t claim the amount within 30 days the bitcoin will be automatically returned to you.

The method below is useful if you would only like to create the user, or would like to send to their bitcoin address instead of an email address.

Automatically Granting Permissions On The Created Account

You can optionally pass in a client_id parameter that corresponds to your OAuth2 application and a scopes parameter which is a space-separated list of permissions. When these are provided, the generated user will automatically have the permissions you’ve specified granted for your application.

Two permissions are not allowed to be automatically, and require explicit user permission. These are the all and send permissions. Other permissions are allowed. For example if you specify the merchant permission, this allows you to create payment buttons, pages, and other merchant tools on the user’s behalf.

An additional oauth field will be returned in the response that contains the necessary fields to perform OAuth requests (access_token, refresh_token, etc). See example below.

Examples

# Request
POST https://api.coinbase.com/v1/users
{
  "user": {
    "email": "newuser@example.com",
    "password": "test123!"
  }
}
# Response
{
  "success": true,
  "user": {
    "id": "501a3d22f8182b2754000011",
    "name": "New User",
    "email": "newuser@example.com",
    "receive_address": "mpJKwdmJKYjiyfNo26eRp4j6qGwuUUnw9x"
  }
}

POST https://api.coinbase.com/v1/users
{

}

{

Params

Param name Description
user
required

User object containing the params


Value: Must be a Hash
user[email]
required

Email address


Value: Must be String
user[password]
required

A strong password - at least eight digits without dictionary words.


Value: Must be String
user[referrer_id]
optional

Optional id of the user to get a referral credit in the case that this transaction makes the user eligible. The referring user is eligible for a credit if the address in the ‘to’ field is an email address for which there is currently no registered account and the recipient proceeds to buy or sell at least 1 BTC.


Value: Must be String
client_id
optional

Optional client_id of your OAuth2 api application. When this is provided your application will automatically be granted the ‘merchant’ OAuth2 permission for this user.


Value: Must be String
scopes
optional

Optional list of authorization scopes your app will obtain for this user, separated by space.


Value: Must be String
